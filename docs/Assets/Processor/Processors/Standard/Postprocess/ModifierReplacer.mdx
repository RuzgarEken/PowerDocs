# Modifier Replacer

The **Modifier Replacer** is the signed version of the **Postprocess Modifier Replacer** Processor.

When a Modifier is about to be added to a Stat, the Processor checks whether a **Modifier Replacer** is defined for that Stat. If the replacement limit has been reached, one existing Modifier is stopped and replaced by the incoming Modifier.

Replacement limits are evaluated separately for the **Source Actor** and the **Target Actor**.

### Target Side

The Processor checks the number of matching Modifiers currently applied to the **Target Stat**.

If the configured replacement limit has been reached, one existing matching Modifier is stopped before the incoming Modifier is applied.

### Source Side

The Processor checks how many matching Modifiers on the **Target Stat** were applied by the **Source Actor**.

For example, suppose the **Poison** Stat has a Source-side replacement limit of **1**.

Even if the Target already has three Poison Modifiers, the incoming Modifier is applied if none of them belong to the current Source Actor. However, if one matching Modifier already belongs to the current Source Actor, that Modifier is stopped and replaced with the incoming Modifier.

The Modifier Replacer Stat is resolved as follows:

### Base Stat (BS) Processors

- For the **Target Actor**, the Modifier Replacer Stat is obtained directly from the **Target Stat's Modifier Replacer Relative Stat**.
- For the **Source Actor**, the Modifier Replacer Stat is obtained from the **Source Actor** using the **Stat Key** referenced by the **Target Stat's Modifier Replacer Relative Key**.

### Attribute Stat (AS) / Multi Attribute Stat (MAS) Processors

- For the **Target Actor**, the Modifier Replacer Stat is obtained from the **Target Actor** using the **Stat Key** referenced by the **Attribute Stat's Modifier Replacer Relative Key**.
- For the **Source Actor**, the Modifier Replacer Stat is obtained directly from the **Attribute Stat's Modifier Replacer Relative Stat**, since the Attribute Stat already belongs to the Source Actor.

The resolved Stat must be an **Int Stat**.
